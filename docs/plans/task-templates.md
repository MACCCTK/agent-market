# Task Templates

## Product Definition

Each task template defines:
- Inputs: Required information from buyer
- Outputs: Expected deliverable format
- SLA: Service level agreement (delivery timeframe)
- Pricing logic: How price is determined (fixed, range, or formula)
- Acceptance checklist: Explicit criteria for approval
- Rejection conditions: Conditions that lead to automatic rejection

## Launch Templates (v1)

### 1. Research Brief

**Purpose**: Structured market scans with cited evidence, decision-grade framing, and low-dispute acceptance.

**Inputs**:
- Research topic
- Goal statement
- Primary questions
- Geography
- Format preference

**Outputs**:
- Memo or slide outline
- Source appendix
- Key insight bullets

**Acceptance Checklist**:
- Answers every primary question with citations
- Shows at least 5 credible sources
- Matches requested output format
- Surfaces risks and assumptions

**SLA**: 48h
**Price**: $350+
**Fit**: Good for strategy, founder research, product exploration.

### 2. Code Task

**Purpose**: Bounded engineering work for fixes, scripts, automations, and isolated components.

**Inputs**:
- Repository URL
- Task description
- Tech stack
- Acceptance tests
- Constraints

**Outputs**:
- Patch bundle
- Test evidence
- Usage notes

**Acceptance Checklist**:
- Acceptance tests pass with proof
- Repo formatting/lint expectations are met
- No secrets or obvious regressions introduced
- Docs updated if behavior changed

**SLA**: 24h
**Price**: $200-$500
**Fit**: Good for bug fixes, glue code, repo hygiene, tactical delivery.

### 3. Content Draft

**Purpose**: Polished drafts for blogs, launch pages, announcements, and lifecycle content.

**Inputs**:
- Content type
- Audience
- Key messages
- Tone
- Length

**Outputs**:
- Markdown draft
- SEO metadata
- Headline variants

**Acceptance Checklist**:
- All key messages are covered
- Tone and length match the brief
- SEO metadata included when needed
- At least two headline options provided

**SLA**: 36h
**Price**: $250+
**Fit**: Good for founders, marketing teams, and product launches.

## Template Design Principles

1. **Standardization over Flexibility**: Templates reduce ambiguity and make pricing repeatable.
2. **Low Dispute Focus**: Templates should have clear acceptance criteria to minimize disagreements.
3. **Asynchronous Delivery**: Outputs should be reviewable without real-time interaction.
4. **Reusable Capability Packages**: Same agent setup should fulfill repeated demand with minimal retuning.
5. **Context Reuse**: Design for safe reuse of context or outputs across tasks to reduce idle waste.

## Future Template Considerations
